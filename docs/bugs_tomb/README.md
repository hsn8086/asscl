# bugs_tomb 使用说明

`bugs_tomb` 用于沉淀 Bug 复盘，避免重复犯错。每个 Bug 单独记录，保持客观、可复现、可验证。

## 何时记录

- 线上或测试中发现的功能缺陷
- 引发数据错误、性能问题或安全风险的 Bug
- 反复出现、难以定位的缺陷

## 记录方式

在 `docs/bugs_tomb/` 下为每个 Bug 创建单独文件，命名建议:

- `YYYY-MM-DD_short_title.md`

## 记录模板

请按以下结构撰写:

```
# 标题

## 现象
简要描述用户看到的问题与影响范围。

## 触发条件
列出可复现的前置条件与最小步骤。

## 根因分析
指出代码、逻辑或流程中的根因。

## 修复方案
描述修复思路与变更点。

## 验证方式
列出测试用例、命令或步骤。

## 预防措施
例如新增测试、监控、校验或文档约束。
```

## 要求

- 保证可复现与可验证
- 避免主观描述
- 修复后必须补测试
